# Copilot Commit Message Instructions

Please generate commit messages that follow the [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) format.

Only include staged files when generating the commit message. Do not include changes from unstaged files or describe unrelated code. Use the last three commit messages as style reference.

The message must be in this format:

<type>(<scope>): <short description>

[optional body]

[optional footer(s)]

Allowed types:
- feat: A new feature
- fix: A bug fix
- docs: Documentation changes
- style: Code style change (formatting, missing semi-colons, etc)
- refactor: Code changes that neither fix a bug nor add a feature
- perf: Performance improvements
- test: Adding or updating tests
- chore: Other changes that don’t modify src or test files
